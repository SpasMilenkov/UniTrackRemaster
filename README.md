## ğŸ—ï¸ Architecture

UniTrack is built on a modern microservices architecture with 8 interconnected services:

```mermaid
graph TB
    subgraph "Frontend"
        UI[Nuxt.js PWA<br/>Vue 3 + TypeScript]
    end
    
    subgraph "Backend Services"
        API[.NET 8 API<br/>2 Replicas]
        DB[(PostgreSQL<br/>Main Database)]
        VDB[(Qdrant<br/>Vector DB)]
        AI[Ollama<br/>AI Models]
    end
    
    subgraph "Infrastructure"
        LB[NGINX<br/>Load Balancer]
        MON[Prometheus<br/>+ Grafana]
    end
    
    UI --> LB
    LB --> API
    API --> DB
    API --> VDB
    API --> AI
    MON --> API
```

### ğŸ› ï¸ Technology Stack

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Frontend** | Nuxt 3, Vue 3, TypeScript | Progressive Web Application |
| **UI Components** | Naive UI, Tailwind CSS | Modern, responsive interface |
| **Backend** | ASP.NET Core 8 | RESTful API with SignalR |
| **Database** | PostgreSQL | Primary data storage |
| **Vector Database** | Qdrant | AI/ML embeddings storage |
| **AI/ML** | Ollama (Llama 3.2) | Local language model inference |
| **Caching** | Redis (in-memory) | Performance optimization |
| **File Storage** | Firebase Cloud Storage | Document and media storage |
| **Monitoring** | Prometheus + Grafana | System observability |

### ğŸ“Š Database Architecture

The system uses **51 database tables** organized into 5 main functional areas:

1. **User & Identity Management** (16 tables)
2. **Academic Structure** (18 tables)
3. **Performance Tracking** (6 tables)
4. **Communication** (7 tables)
5. **Analytics & Reporting** (4 tables)
